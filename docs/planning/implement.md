# Trellis 可视化管理项目实施清单

## 实施边界

- 实现目标位于独立仓库，不修改当前 Trellis CLI 的发布结构。
- 首版只读；任何对被查看项目 `.trellis/` 的写入都属于越界行为。
- 首版不引入 `@mindfoldhq/trellis-core` 强制依赖；只有发现任务语义复用确有收益时再增加可选适配。

## 阶段 1：独立仓库与运行骨架

- [ ] 初始化独立 pnpm/TypeScript 项目。
- [ ] 建立 `src/server`、`src/web`、`src/shared` 边界。
- [ ] 配置 React + Vite Web 构建和 Node 本地服务启动命令。
- [ ] 开发模式同时启动服务与 Vite，并自动打开浏览器；生产模式由 Node 服务托管静态资源。
- [ ] 服务默认绑定 `127.0.0.1`，实现基础健康检查接口。

## 阶段 2：本机注册表与摘要快照

- [ ] 定义项目注册表结构：稳定 ID、路径、显示名称、状态、焦点标记、索引时间和错误信息。
- [ ] 使用操作系统应用数据目录保存注册表，不写入 Trellis 项目。
- [ ] 定义摘要快照结构：项目概览、Spec 树、Task 摘要、Workflow 摘要、索引时间和诊断信息。
- [ ] 实现注册表/快照的安全读写、损坏恢复和版本字段。
- [ ] 验证删除应用数据不会修改源项目。

## 阶段 3：项目扫描与 Trellis 内容解析

- [ ] 实现用户指定目录下的递归扫描，跳过 `.git`、`node_modules`、构建产物、缓存和符号链接逃逸路径。
- [ ] 实现单项目手动添加和 `.trellis/` 结构校验。
- [ ] 解析项目概览、monorepo 包信息、Workflow 摘要、Spec 文件树和 Task 摘要。
- [ ] 对 `task.json`、Markdown、JSONL、YAML 缺失或格式异常返回带源路径的诊断，不让单个坏文件阻塞整个项目。
- [ ] Markdown 正文按需读取，历史项目默认只使用摘要快照。

## 阶段 4：焦点项目与实时更新

- [ ] 实现 `history`、`focus`、`unavailable` 生命周期。
- [ ] 焦点项目进入时先执行索引，成功后启动监听。
- [ ] 只监听焦点项目的 `.trellis/spec/`、`.trellis/tasks/` 和必要配置文件。
- [ ] 使用原生文件事件封装，加入防抖、批量失效、路径归一化和监听错误处理。
- [ ] 项目移出焦点时停止并释放监听器，保留最后摘要快照。
- [ ] 实现 SSE 事件：项目聚焦、项目失效、Spec 变化、Task 变化、项目不可用和重新索引完成。
- [ ] 应用重启时恢复焦点集合，逐个重新索引并恢复监听；历史项目不启动监听。
- [ ] 原生监听不可用时允许低频快照降级，并在接口和页面标记非实时状态。

## 阶段 5：只读 HTTP API 与 Web UI

- [ ] 提供项目列表、快速扫描候选、手动添加、焦点切换和项目刷新 API；写入范围仅限应用自己的注册表。
- [ ] 提供项目概览、Spec 树、Spec 正文、Task 列表、Task 详情、Workflow 摘要和诊断 API。
- [ ] 提供 SSE 订阅接口，事件只包含项目 ID、资源类型、失效范围和时间戳，不把大段 Markdown 塞入事件。
- [ ] 实现“焦点项目 / 历史项目”分组导航和状态展示。
- [ ] 实现项目概览、Spec 阅读、Task 活动/归档、Task 详情、Workflow 摘要页面。
- [ ] 提供源文件路径、最后索引时间、可能过期和非实时状态提示。
- [ ] 提供外部文件管理器/编辑器打开入口，但不提供应用内编辑控件。

## 阶段 6：验证与交付

- [ ] 用临时 fixture 项目验证快速扫描、手动添加、无效项目和重复项目处理。
- [ ] 验证焦点项目文件变化能触发局部刷新，历史项目变化不会触发监听事件。
- [ ] 验证应用重启恢复焦点项目，监听器数量与焦点数量一致。
- [ ] 验证项目移动、删除、权限异常、坏 JSON/Markdown 和监听失效的诊断表现。
- [ ] 运行 `pnpm lint`、`pnpm typecheck`、`pnpm build`。
- [ ] 运行生产模式启动命令，确认浏览器打开、静态资源加载和服务退出清理。
- [ ] 记录监听目录数、扫描耗时、事件队列长度、缓存大小等运行指标，为多项目基准测试留出入口。

## 风险与回滚点

- 文件扫描和路径校验是最高风险区域；任何路径越界都必须阻止并记录诊断。
- 监听器生命周期是资源风险区域；焦点切换和服务退出必须可观测、可释放。
- 历史摘要不能被误认为源数据；页面必须显示索引时间和快照状态。
- 若跨平台原生监听库在目标环境不稳定，可先切换到轮询降级，不改变 API 和 UI 合同。
- 若后续发现任务状态/阶段规则与 Trellis 漂移，再单独评估接入 Core SDK，不反向扩大首版范围。

## 当前不做

- 不创建测试类，除非用户另行要求；优先完成手工 fixture 验证和类型/构建质量门禁。
- 不实现任务、规范、Workflow 的编辑或状态操作。
- 不实现团队、远程、云端、桌面安装、Channel、Mem、Workspace 和 Runtime 面板。
