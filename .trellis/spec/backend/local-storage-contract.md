# 本机存储可执行合同

## 范围

修改应用数据目录、`registry.json`、`snapshots.json`、版本、迁移、原子写或清理时必须遵守本合同。源项目 `.trellis` 不是应用存储。

## 当前合同

- 固定文件为 `<应用数据目录>/registry.json` 和 `snapshots.json`，当前根版本为 2。
- macOS 目录为 `~/Library/Application Support/Trellis Visual Console`。
- Windows 目录为 `%APPDATA%\Trellis Visual Console`；`APPDATA` 缺失时回退用户 Roaming 目录。
- `TRELLIS_VISUAL_CONSOLE_DATA_DIR` 仅用于开发和隔离验证；相对值先解析为绝对路径。
- 注册表与快照的完整读改写由一个互斥锁串行保护。
- 保存使用 UTF-8、同目录临时文件、文件同步和原子替换。
- 未来版本必须停止初始化并保留原字节，禁止当作损坏恢复。
- v2 损坏数据隔离为 `.corrupt-*`；恢复行为不得影响版本不兼容文件。

## 版本 1 迁移

1. 完整读取并严格校验 v1 注册表；失败时不写任何文件。
2. 原字节备份注册表和快照，备份名不覆盖既有文件。
3. 写入 v2 空快照，旧快照只备份、不作为运行数据。
4. 最后提交 v2 注册表，保留项目 ID、路径、名称、状态和时间。
5. 缺失快照项目在后台重建；中断重启必须幂等继续。

## 清理

清理只能删除固定应用数据目录中的注册表、快照和日志。不得遍历、修改或删除已登记项目。

## 验证

临时目录至少覆盖首次创建、v1 双备份、迁移中断恢复、损坏隔离、未来版本保护、并发保存、日志共存和源项目零写入。
