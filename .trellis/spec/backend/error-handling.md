# 后端错误处理

## 当前约定

- Core 对外错误使用 `CommandError { code, message, details }`，消息为稳定中文。
- 不返回绝对项目路径、堆栈、底层 I/O/Serde/notify 原文或平台错误码。
- 初始化失败存入 `AppState`，窗口继续显示；业务 Command 返回可恢复初始化错误。
- 未来存储版本停止初始化并保留原文件，不能降级或覆盖。
- 单文件解析错误进入项目诊断，不中断其他文件或项目。
- 路径越界统一返回稳定安全错误，禁止捕获后读取原始输入。
- native watcher 失败明确降级；polling 失败进入 `stopped`，不能伪装实时。
- 项目重新校验失败进入 `unavailable`、释放 watcher、保留最后快照。
- 对话框取消是正常响应；并发对话框返回稳定忙碌错误。
- 清理即使 Core 初始化失败也只处理固定应用数据目录。

## 禁止模式

- 在 Command adapter 中拼接业务错误文本或泄漏路径。
- 把 HTTP 状态码、Node 错误或 Tauri 插件原始错误作为前端合同。
- 吞掉 watcher、存储或关闭失败后继续报告成功。
- 用空默认文件覆盖不兼容版本或迁移失败数据。
