# 阶段五只读 HTTP API 与 Web UI

## 目标

把阶段二至阶段四建立的注册表、摘要快照、项目内容读取和实时事件能力交付为可直接使用的本地只读产品：用户可以发现和登记 Trellis 项目，在焦点/历史分组中查看项目状态，浏览概览、Spec、Task、Workflow 与诊断，并在焦点项目变化后通过 SSE 自动刷新页面。

## 背景与已确认事实

- 服务固定绑定 `127.0.0.1`，浏览器和服务同源访问，不增加登录、远程访问或 CORS 配置。
- 项目注册表与快照位于应用数据目录；源项目 `.trellis/` 是唯一事实来源，接口只允许写应用自己的注册表和快照。
- `ProjectCatalog` 已提供扫描、登记、刷新和状态读取基础；`ProjectRealtimeManager` 已提供聚焦、取消聚焦、运行时状态和事件中心。
- 历史项目启动时不访问文件系统；打开历史项目时可以显式刷新一次，但不会自动启动监听。
- 浏览器不能可靠提供用户目录的本机绝对路径，首版扫描和手动添加界面使用带说明的路径文本输入，不引入 Electron/Tauri、平台脚本或原生目录选择器。
- 阶段四事件合同已经位于 `src/shared/project-events.ts`，SSE 路由和 Web UI 必须直接复用，不重新定义事件字段。
- UI 采用信息密集、克制的开发者控制台风格：深色 Slate 基底、单一绿色状态强调、Inter/系统字体、清晰网格和高对比度，不使用霓虹、装饰性动画或 Emoji 图标。
- 本阶段不生成测试文件，使用临时 fixture、HTTP 请求和浏览器流程完成验收。

## 需求

### 共享 API 合同

1. 在 `src/shared` 单点定义项目列表、项目详情、扫描、登记、操作结果、Spec 文档、Task 详情/文档和错误响应的运行时 Schema、TypeScript 类型与解析入口。
2. Web UI 对所有 JSON 响应先按 `unknown` 接收，再通过共享 Schema 校验；禁止组件内直接断言接口类型。
3. 日期时间使用 ISO 8601 字符串；接口不返回本机堆栈、任意文件内容或未登记项目数据。

### 项目与内容 API

4. 提供项目列表 API，合并注册表、最后快照和运行时监听状态，返回焦点/历史/不可用状态、最后索引时间、可能过期、实时/轮询状态和错误摘要。
5. 提供快速扫描 API，接收显式扫描根路径，返回未持久化候选和扫描诊断；扫描操作本身不得写注册表。
6. 提供单个或批量登记 API，只将通过服务端校验的项目写入应用注册表和快照。
7. 提供项目详情 API，返回已登记项目、运行时状态和最后摘要快照；历史项目只读快照，不因 GET 请求访问源文件。
8. 提供焦点切换 API：加入焦点时复用“先刷新、后监听”流程；移出焦点时释放监听并保留快照。
9. 提供项目刷新 API，允许用户对历史、焦点或不可用项目发起一次显式重新校验和索引。
10. 提供 Spec Markdown 正文 API；请求路径必须先出现在该项目快照的 Spec 树中，再通过现有 `.trellis` realpath 边界读取。
11. 提供 Task 详情 API，根据快照中的任务 `sourcePath` 定位任务目录，列出 Markdown 和 JSONL 文档；不接受任意任务目录路径。
12. 提供 Task 文档 API，只读取任务详情已列出的 Markdown/JSONL 文件，使用严格 UTF-8 并阻止符号链接、目录穿越和任务目录逃逸。
13. 项目详情中的快照直接提供项目概览、Spec 树、活动/归档 Task 列表、Workflow 摘要和诊断，不额外复制同一套数据模型。
14. 提供外部打开 API：空 `sourcePath` 打开已登记项目目录；非空路径只能位于该项目 `.trellis/` 内并通过路径边界校验，再交给系统默认文件管理器或应用打开。接口不接受绝对路径、命令或启动参数。

### SSE

15. 提供 `GET /api/events` SSE 订阅，使用 `text/event-stream`、禁用缓存并保持连接。
16. 每条 SSE 数据直接序列化 `ProjectRealtimeEvent`，包含事件 ID、项目 ID、资源、失效范围、时间戳和监听模式，不包含 Markdown 或完整快照。
17. SSE 连接关闭时立即取消事件订阅和心跳定时器；服务退出不能遗留连接资源。

### Web UI

18. 实现焦点项目、历史项目和不可用项目的分组导航，项目行显示状态文字、监听模式和最后索引时间，当前选择有明确活动态。
19. 实现项目发现面板：快速扫描根路径、候选多选批量登记、单项目路径手动添加，并对无效路径和扫描诊断提供中文反馈。
20. 实现项目级操作：刷新、加入/移出焦点、打开项目目录；操作期间禁用重复提交并显示成功或失败结果。
21. 实现项目概览页面，展示路径、状态、包信息、Workflow 摘要、活动/归档任务数量、诊断数量、最后索引时间和实时性。
22. 实现 Spec 页面：保留目录树，点击 Markdown 文件后按需加载并安全渲染正文；显示源路径和修改时间。
23. 实现 Task 页面：活动/归档切换、任务列表、任务元数据、文档清单和按需正文阅读；Markdown 安全渲染，JSONL 使用可滚动等宽文本展示。
24. 实现 Workflow 页面，展示名称、当前阶段、阶段说明、源路径和快照时间。
25. 实现诊断页面或诊断区域，按错误/警告展示代码、中文消息和项目相对源路径；项目不可用时提供刷新恢复入口。
26. 页面通过 `EventSource` 接收事件；选中项目受影响时刷新项目列表和详情，当前 Spec/Task 文档按资源类型重新读取。
27. 使用 URL 查询参数保存当前项目、视图和选中文档，刷新页面后尽量恢复上下文，不引入路由库。
28. 提供清晰的加载、空、错误和重试状态；错误使用 `role="alert"` 或 `aria-live`，所有点击行为使用语义化 `button`。
29. 使用 Lucide 统一 SVG 图标；交互具备可见键盘焦点、150–250ms 颜色过渡、`cursor: pointer`，并尊重 `prefers-reduced-motion`。
30. 响应式覆盖 375px、768px、1024px 和 1440px；窄屏允许导航与内容纵向堆叠，禁止水平页面滚动。
31. Markdown 默认忽略原始 HTML，不接入 `rehype-raw`，避免不安全 HTML 注入。
32. 所有新增方法和组件添加中文注释，关键接口校验、SSE 清理和路径边界添加中文说明。

## 验收标准

- [x] 项目列表能正确区分焦点、历史和不可用项目，并显示 `native | polling | stopped` 与最后索引时间。
- [x] 快速扫描不写注册表，用户选择候选后可以批量登记；重复项目不会产生重复记录。
- [x] 手动添加有效项目成功，无效路径返回可恢复的中文错误与诊断。
- [x] 历史项目 GET 详情只读取快照；用户选择历史项目时可以显式刷新，但不会创建监听器。
- [x] 加入焦点后监听器启动并显示实时/轮询状态，移出焦点后监听器释放且快照仍可浏览。
- [x] Spec 树和 Markdown 正文可浏览，非法路径、非 Spec 路径和链接逃逸均被拒绝。
- [x] 活动/归档 Task 可切换，任务规划文档和 JSONL 清单可按需阅读。
- [x] 项目概览显示包、Workflow、任务数、诊断数、源路径和最后索引时间。
- [x] SSE 只发送轻量事件；焦点项目文件变化后当前页面自动刷新，连接关闭后订阅被释放。
- [x] 页面明确标记历史快照可能过期、轮询非实时和项目不可用状态。
- [x] 项目目录和合法 `.trellis` 源文件可以交给系统外部应用打开，任意绝对路径或越界路径被拒绝。
- [x] Markdown 原始 HTML 不会作为 DOM 执行或渲染。
- [x] 所有接口错误不暴露堆栈，未知 `/api/*` 仍返回统一 404。
- [x] 页面在 375px、768px、1024px、1440px 下无水平滚动，键盘焦点和错误提示可见。
- [x] 浏览、刷新、聚焦和外部打开流程不会修改被查看项目的 `.trellis/` 文件。
- [x] 没有引入数据库、WebSocket、路由库、全局状态库、CSS 框架、Core SDK 或桌面壳。
- [x] `pnpm lint`、`pnpm typecheck`、`pnpm build` 全部通过。

## 验收记录

- 临时 fixture 验证扫描、登记、焦点切换、刷新、Spec/Task 白名单、JSONL、外部打开拒绝与 SSE 轻量事件。
- Playwright 验证项目发现、中文错误、URL 失效选择回退、SSE 自动刷新、Markdown 原始 HTML 安全和四档响应式布局。
- 显式刷新、加入焦点、移出焦点前后，fixture `.trellis` 文件摘要保持一致。

## 不在本阶段范围内

- 浏览器原生目录选择器、macOS/Windows/Linux 平台脚本或桌面壳。
- 删除已登记项目、修改显示名称或维护扫描根目录历史。
- 应用内编辑 Markdown、JSON、JSONL、YAML、Task 状态或 Workflow。
- 搜索、全文索引、图表、导出、主题切换和用户自定义布局。
- 语法高亮、Mermaid 渲染、不安全 HTML 和远程图片代理。
- Workspace、Journal、Mem、Channel、Runtime、团队、账号和远程访问。
- 阶段六的完整跨平台、权限、性能和多项目基准测试。
