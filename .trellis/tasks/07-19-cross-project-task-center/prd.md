# 跨项目任务中心与搜索筛选

## Goal

在不逐个切换项目的情况下，集中查看本机已登记 Trellis 项目的任务状态，并通过搜索、筛选和排序快速定位需要关注的任务。

## Background

- 当前控制台以单项目工作区为中心，Task 页面只展示当前项目的活动与归档任务。
- 项目列表 API 已提供各项目的活动任务数、归档任务数和诊断数，但没有跨项目任务明细查询入口。
- 当前任务摘要已经包含 `id`、标题、状态、阶段、负责人、包、更新时间、父子关系和源路径，可作为第一阶段聚合与筛选的数据基础。
- 现有 Task 页面已经支持父子层级、活动/归档切换、任务详情、规划文档、URL 恢复和外部打开，本任务必须复用这些能力，不重新实现第二套任务详情页。
- 历史任务规划曾明确把任务搜索、状态筛选、状态汇总和全文索引放在首版范围之外；本任务是对该延期能力的正式补充。
- 产品继续保持个人本机、只读和源项目零写入边界，不增加团队账号、远程访问或任务状态修改能力。

## Requirements

### R1 跨项目任务中心与焦点范围

- 提供独立于单项目工作区的跨项目任务中心，默认聚合焦点项目中存在可用快照的任务。
- 提供“焦点项目 / 全部项目”范围切换；切换到全部项目后，聚合所有已登记且存在可用快照的项目任务。
- 全部项目范围中，焦点项目任务优先于历史项目任务展示；同一项目状态内再应用用户选择的排序规则。
- 历史项目任务必须明确标记为历史快照，并展示项目最后索引时间，不能伪装成实时数据。
- 不可用项目默认不混入任务结果，只在任务中心汇总区域展示数量并提供诊断入口。
- 跨项目结果采用适合搜索、筛选和排序的扁平工作列表，不在任务中心重复单项目任务树。
- 子任务在列表中展示父任务名称作为上下文；完整父子树、进度和祖先导航继续由现有单项目 Task 页面负责。
- 每条任务至少展示所属项目、标题、活动/归档集合、状态、阶段、负责人、包和更新时间。
- 不可用项目或没有快照的项目不能阻塞其他项目展示，并应提供可理解的状态提示。
- 点击任务后复用现有项目 Task 详情页，并恢复对应项目、任务和默认文档的 URL 状态；默认文档按 `prd.md`、`design.md`、`implement.md` 优先，其余文档按路径排序选择首个存在项，无文档时保持未选择状态。

### R2 搜索、筛选与排序

- 关键词搜索覆盖项目名称、任务标题、任务 ID、状态、阶段、负责人和包，不读取 PRD、设计、实施计划或 JSONL 正文。
- 关键词搜索同时匹配状态原始值和页面中文展示值；匹配忽略大小写并采用包含匹配，空白关键词等同于未搜索。
- 任务中心默认只展示活动任务，并提供“活动 / 归档 / 全部”集合筛选。
- 支持按所属项目、任务集合、状态、阶段、负责人和包组合筛选。
- 标准状态固定为 `planning / in_progress / review / completed`，旧状态 `done` 在筛选和汇总中归入 `completed`；未知状态在筛选项中保留独立原始值，在状态汇总中统一计入“其他”，保证状态数量之和与结果总数闭合。
- 支持按更新时间排序，默认优先展示最近更新任务；排序结果在筛选变化后保持确定性。
- 项目范围、搜索、筛选和排序状态应进入 URL，使刷新和分享本机深链后能够恢复。
- 非法 URL 枚举值必须回退默认值；不存在或已过期的项目、状态、阶段、负责人和包筛选必须自动清除，并将修复后的有效状态重新写回 URL。
- 无匹配结果时保留当前筛选条件，并提供清晰空状态和一键清除入口。

### R3 汇总与现有能力兼容

- 任务中心展示当前结果总数，并提供活动、归档及各标准状态的汇总数量。
- 汇总数量应基于当前项目范围、搜索与筛选结果计算，不能与列表实际内容不一致。
- 单项目 Task 页面原有层级、进度、详情、文档阅读、外部打开和 URL 恢复行为不得回归。
- 前后端共享 DTO 继续在 `src/shared` 单点定义，未知 API 响应仍需经过运行时 Schema 校验。

## Acceptance Criteria

- [ ] 用户进入任务中心时默认看到焦点项目任务，并可以切换到全部项目范围。
- [ ] 全部项目范围中，焦点任务优先展示，历史任务带有快照标识和最后索引时间。
- [ ] 不可用项目不混入任务列表，但汇总区域可以看到数量并进入诊断。
- [ ] 跨项目任务使用扁平列表展示，子任务可以识别所属父任务，筛选结果不会因缺少祖先节点而不可理解。
- [ ] 用户进入任务中心时默认只看到活动任务，并可以切换到归档或全部任务。
- [ ] 用户可以组合使用项目、集合、状态、阶段、负责人和包筛选任务。
- [ ] 用户可以搜索项目名称、任务标题、任务 ID、状态、阶段、负责人和包，并按更新时间排序。
- [ ] 状态搜索能够命中原始值和中文展示值，`done` 与 `completed` 在筛选和汇总中均表现为“已完成”。
- [ ] 标准状态汇总固定展示规划中、实施中、评审中、已完成和其他，数量之和与当前结果总数一致。
- [ ] 首版搜索不会读取或索引 PRD、设计、实施计划和 JSONL 正文。
- [ ] 项目范围、搜索、筛选和排序状态可以通过 URL 恢复。
- [ ] 非法或过期 URL 参数会自动恢复为有效状态，并通过替换当前历史记录写回修复后的 URL。
- [ ] 汇总数量始终与当前结果一致，无结果时可以一键清除条件。
- [ ] 点击跨项目任务可以进入对应项目的现有 Task 详情，按既定优先级选择默认文档，并在最终 URL 中恢复 `project`、`task` 和存在时的 `document` 参数。
- [ ] 不可用项目、无快照项目和损坏任务不会阻塞其他项目任务中心。
- [ ] 现有单项目 Task 浏览、父子层级、文档阅读和外部打开行为无回归。
- [ ] 页面在 375px、768px、1024px 和 1440px 下无页面级水平滚动或内容遮挡。
- [ ] `pnpm lint`、`pnpm typecheck`、`pnpm build` 和 `git diff --check` 全部通过。

## Out of Scope

- 创建、编辑、移动、归档任务或修改任务状态。
- 团队排期、工时统计、甘特图和任务依赖图。
- 团队账号、远程访问、云端同步和跨机器共享。
- 为搜索引入远程服务或数据库服务器。
- PRD、设计、实施计划、研究资料和 JSONL 的全文搜索或持久化内容索引。
