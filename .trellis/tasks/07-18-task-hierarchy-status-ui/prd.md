# 优化任务层级与状态展示

## Goal

优化只读控制台的 Task 浏览体验，让用户能快速理解任务之间的父子关系、当前状态与推进情况，减少在平铺列表中逐项打开 `task.json` 才能判断任务结构的成本。

## Background

- 当前 Task 页面仅按“活动 / 归档”分组，并在组内按更新时间倒序平铺展示，见 `src/web/components/TaskBrowser.tsx`。
- 当前所有任务均使用同一个 `CircleDot` 图标，状态主要依赖“规划中 / 实施中 / 已完成”文本表达。
- `TaskSummaryApi` 和持久化快照当前没有父子关系字段，见 `src/shared/api.ts` 与 `src/server/storage/models.ts`。
- Trellis `task.json` 已原生提供 `parent`、`children`、`subtasks` 字段；`task.py list` 也会将无父任务的节点作为根节点并缩进展示子任务。
- Trellis CLI 的状态过滤说明包含 `planning / in_progress / review / completed`，但当前页面只格式化前三种状态，`review` 会直接显示原始英文值。
- 当前仓库中的既有任务暂未形成真实父子样例，因此实现与验收需要构造覆盖活动、归档和跨集合关系的 fixture。

## Requirements

### R1 任务层级

- Task 页面应展示 `parent` 与 `children` 所表达的父子关系，不能再把所有任务无差别地平铺。
- 列表采用可折叠树形交互：父任务提供明确的展开 / 收起控制，子任务通过缩进与连接层级展示。
- 默认展开包含未完成子任务的父任务；用户可按父任务独立控制展开状态。
- 折叠状态只在当前页面会话内保留，不写入 URL 或本地持久化；活动与归档 Tab 分别记忆各自的折叠状态。
- URL 恢复或外部导航选中子任务时，必须自动展开该任务的全部可解析祖先，确保选中项在列表中可见。
- 逻辑层级不设上限，但列表视觉缩进最多三级；更深层级继续通过树线和层级标记表达，避免持续压缩任务标题。
- 任务详情区使用可点击面包屑显示当前任务的完整父级路径；点击祖先任务时自动切换到其所属 Tab、展开必要层级并选中该任务。
- 保留“活动 / 归档”两个 Tab；Tab 上的数量只统计各自集合中的实际任务数，不因父子关系重复计数。
- 活动父任务展开后直接展示活动子任务；若存在已归档子任务，则展示归档子任务数量入口，点击后切换到归档 Tab 并定位相应任务。
- 归档 Tab 中的子任务应显示所属父任务名称，避免跨集合后丢失层级上下文。
- 一个父任务存在多个归档子任务时，点击归档入口后切换到归档 Tab，并临时只展示该父任务关联的归档子任务。
- 归档聚焦态在列表顶部显示来源父任务名称和清除按钮；清除后恢复完整归档任务树。该聚焦态只保留在当前页面会话，不写入 URL。
- 层级数据必须由服务端索引和共享 API 合同提供，前端不得通过目录名猜测关系。
- 对缺失父任务、重复引用、循环引用等损坏关系应安全降级，并产生可诊断结果，不能导致整个 Task 页面不可用。

### R2 状态展示

- 明确区分 `planning / in_progress / review / completed` 四种标准状态，并为未知状态保留可读回退。
- 状态表达不能只依赖颜色；应同时具备文字、图标或形状语义。
- 任务行采用“小型状态图标 + 中文状态文字”的紧凑表达，不使用会明显挤压标题宽度的状态胶囊。
- 标准状态图标语义分别为：`planning` 空心圆、`in_progress` 进行图标、`review` 查看图标、`completed` 勾选圆；未知状态使用问号图标并显示原始状态值。
- 父任务显示全部直接子任务的完成进度，采用“已完成数 / 子任务总数”的紧凑表达；统计范围跨活动与归档两个集合。
- 子任务被归档或状态为 `completed` 时计入完成数；统计规则不得受当前选中 Tab 影响。
- 本次不增加按状态筛选、搜索或状态汇总面板。

### R3 保持现有能力

- 保留活动 / 归档任务浏览、任务详情、文档标签、Markdown/JSONL 阅读、URL 恢复和外部打开能力。
- 保持只读原则，不在页面内修改任务关系或状态。
- 在 375px、768px、1024px、1440px 下不得出现页面级水平滚动。

## Technical Notes

- 预计涉及索引快照模型、`TrellisIndexer`、共享 API Schema、API 投影、Task 页面与样式，属于跨层改动。
- 活动任务的子任务可能已经归档；关系解析不能只查看单个集合。
- Trellis CLI 实际以父任务的 `children` 和子任务的 `parent` 维护双向关系；`subtasks` 仅在新建任务时初始化，当前未参与任务链接和进度统计，因此本功能不把它作为层级事实来源。
- 子任务归档后仍保留在父任务 `children` 中，CLI 将不在活动集合中的子任务视为完成；父任务归档时会清空仍存在子任务的 `parent`，索引器需容忍跨集合关系的不对称性。
- 兼容 Trellis 旧状态 `done`：进度统计按已完成处理；页面使用“已完成”语义展示，但标准状态合同仍为 `planning / in_progress / review / completed`。

## Acceptance Criteria

- [x] 有父子关系的任务可从列表中直观看出层级，父任务与直接子任务不会被误认成两个无关任务。
- [x] 父任务可独立展开或收起；包含未完成子任务的父任务首次展示时默认展开。
- [x] 同一页面会话内切换 Tab 后折叠状态仍保留；刷新页面不要求恢复手动折叠偏好。
- [x] 通过 URL 恢复深层子任务时，其全部祖先自动展开且选中项可见。
- [x] 四层及以上任务仍可逐层展开和选择，标题保有可读宽度，详情区显示完整父级路径。
- [x] 详情面包屑可导航到任一可解析祖先任务，并正确同步 Tab、树展开状态、任务选择和文档回退。
- [x] “活动 N / 归档 N”分别等于各集合实际任务数，父任务与子任务不会因树形展示被重复计数。
- [x] 活动父任务能够显示跨集合的直接子任务完成进度，并可通过归档子任务入口切换到归档 Tab 查看对应任务。
- [x] 多个归档子任务可作为一组被聚焦查看，页面明确显示来源父任务，并可一键清除聚焦恢复完整归档列表。
- [x] 归档子任务显示所属父任务上下文。
- [x] `planning / in_progress / review / completed` 均显示稳定的中文状态语义，未知状态不会破坏布局。
- [x] 每种标准状态都有不同图标和中文文字；未知状态显示回退图标与原始状态值。
- [x] 状态不只依赖颜色表达，键盘和屏幕阅读器用户可以识别。
- [x] 父子关系异常时页面仍可浏览，并在诊断区域提供可定位的问题信息。
- [x] 活动 / 归档切换、任务详情、文档阅读、URL 恢复和外部打开行为无回归。
- [x] 375px、768px、1024px、1440px 下无页面级水平滚动，长标题、长状态和深层任务不遮挡其他内容。

## 验收记录

- 临时索引 fixture 验证活动/归档跨集合关系、双向关系优先级、无序目录输入、原始 `children` 顺序、缺失/重复/自引用/循环/多父冲突诊断和旧快照默认值。
- Playwright 合成 DTO 验证 5 层任务树、四类标准状态与未知状态、跨集合 `2/3` 进度、归档子任务聚焦和父级面包屑导航。
- Playwright 在 375px、768px、1024px、1440px 下验证页面滚动宽度等于视口宽度、任务行无越界，并人工复核 375px 与 1440px 截图。
- 独立深层 URL 验证 4 个祖先自动展开、选中项可见、4 级面包屑正确，以及切换活动/归档 Tab 后手动折叠状态保持。
- 最终 `pnpm lint`、`pnpm typecheck`、`pnpm build`、`git diff --check` 通过。

## Out of Scope

- 在控制台内创建、编辑、移动、归档任务或修改状态。
- 甘特图、依赖关系图、团队排期和工时统计。
- 按任务状态筛选、任务搜索和状态汇总面板。
- 改造 Trellis 的任务文件格式或 CLI 状态机。
