# 阶段四焦点项目与实时更新

## 目标

在现有项目注册表、摘要快照和 Trellis 索引能力之上，建立焦点项目的完整运行时生命周期：用户显式聚焦项目后先刷新摘要，再持续监听 Trellis 展示范围内的文件变化并发布轻量失效事件；历史项目保持零监听；项目失效、应用重启和服务退出时都能正确恢复或释放资源。

## 背景与已确认事实

- 注册表已经持久化 `history | focus | unavailable` 状态，摘要快照与源项目完全分离。
- `ProjectCatalog` 已支持项目校验、完整索引、登记和快照保存，但尚未提供已登记项目刷新与状态迁移接口。
- 首版只读，任何生命周期、监听和事件处理都不得写入被查看项目的 `.trellis/`。
- 只需要观察 `.trellis/spec/`、`.trellis/tasks/`、`.trellis/config.yaml` 和 `.trellis/workflow.md`；源码目录和扫描根目录不进入长期监听范围。
- 普通本地磁盘使用操作系统原生文件事件；网络盘、容器挂载等不可靠环境允许降级为低频轮询，并明确标记为非实时。
- 阶段五负责正式 HTTP 查询接口、SSE 订阅路由和 Web UI；本阶段提供可直接接入 SSE 路由的事件合同、订阅中心和运行时状态查询能力。
- Chokidar 5 支持当前 Node.js 版本，依赖规模小于带原生二进制的 `@parcel/watcher`，并同时支持原生事件与轮询模式，适合作为首版监听适配器。

## 需求

1. 为已登记项目实现显式聚焦、移出焦点、重新索引、项目不可用和启动恢复流程。
2. 项目进入焦点时必须先校验并完成一次索引；只有索引成功后才能启动监听并持久化 `focus` 状态。
3. 项目移出焦点时停止并释放监听器，持久化为 `history`，保留最后一次成功摘要快照和索引时间。
4. 项目路径不存在、权限不足或 Trellis 结构失效时，停止监听、持久化为 `unavailable`、保留最后成功快照并记录错误摘要。
5. 应用启动时只恢复注册表中的 `focus` 项目：逐个重新校验、索引并启动监听；`history` 和 `unavailable` 项目不主动访问文件系统。
6. 服务退出时关闭全部项目监听器和待处理定时器，不遗留后台资源。
7. 监听范围严格限制为 `.trellis/spec/`、`.trellis/tasks/`、`.trellis/config.yaml` 和 `.trellis/workflow.md`，禁止跟随符号链接或扩展到项目源码。
8. 文件事件统一转换为规范化的项目内相对路径，经过短时间防抖和批量合并后触发一次项目级重新索引；编辑器一次保存产生的多事件不能造成重复刷新风暴。
9. 根据变化路径发布 Spec、Task 或项目级失效事件；每批变化完成索引后发布重新索引完成事件。
10. 定义轻量事件合同，覆盖 `project-focused`、`project-invalidated`、`spec-changed`、`tasks-changed`、`project-unavailable` 和 `project-reindexed`，事件只包含事件 ID、项目 ID、资源类型、失效范围、时间戳和监听模式，不携带 Markdown 正文或完整快照。
11. 提供进程内事件订阅中心，支持多个订阅者注册和取消订阅，为阶段五 SSE 路由直接复用。
12. 原生监听启动或运行失败时自动切换为低频轮询模式；轮询模式继续触发相同的失效和重新索引流程，并通过运行时状态及事件标记 `polling`，不得伪装为实时。
13. 同一项目的聚焦、取消聚焦、重新索引和降级切换必须串行执行；不同项目对注册表和快照的读改写也不得互相覆盖。
14. 所有新增方法添加中文注释，关键生命周期、只读边界、批处理和降级流程添加中文说明。
15. 本阶段不生成测试文件，使用临时 fixture 和可注入的防抖/轮询参数完成验证。

## 验收标准

- [x] 历史项目在应用启动后没有活动监听器，也不会被主动校验或重新索引。
- [x] 聚焦历史项目时先生成新快照，再进入 `focus` 并启动一个项目级监听器。
- [x] 修改焦点项目的 Spec、Task、配置或 Workflow 文件后，约一秒内只触发一批重新索引，并发布对应失效事件和 `project-reindexed`。
- [x] 修改历史项目文件不会产生监听事件或快照写入。
- [x] 项目移出焦点后监听器数量立即减少，最后成功快照仍可加载，状态变为 `history`。
- [x] 焦点项目被移动、删除、失去权限或结构失效后变为 `unavailable`，监听器释放，最后快照保留并发布 `project-unavailable`。
- [x] 应用重启后只恢复持久化的焦点集合，监听器数量与成功恢复的焦点项目数量一致。
- [x] 模拟原生监听启动失败时可以进入低频轮询模式，运行时状态和事件均标记为非实时。
- [x] 多个快速文件事件会被规范化、去重和批量处理，不会并发覆盖注册表或快照。
- [x] 事件载荷不包含 Markdown 正文、完整 Spec 树、完整 Task 列表或任意绝对源路径。
- [x] 服务退出时所有监听器、轮询器和防抖定时器均已关闭。
- [x] 没有新增正式项目查询 API、SSE HTTP 路由、Web UI 页面或 Core SDK 依赖。
- [x] `pnpm lint`、`pnpm typecheck`、`pnpm build` 全部通过。

## 不在本阶段范围内

- 项目列表、聚焦切换、刷新和内容查询 HTTP API。
- `/api/events` 等正式 SSE 订阅路由及浏览器 `EventSource` 接入。
- 焦点/历史导航、实时状态徽标和内容页面。
- 增量解析单个 Markdown 文件；首版在批次失效后重建项目摘要以保证最终一致。
- 监听项目源码、扫描根目录或所有已登记历史项目。
- 任务、规范、Workflow 编辑和任何源项目写操作。
- Watchman、数据库、Core SDK、桌面安装包和团队能力。
