# 阶段六验证与交付

## 目标

对阶段一至阶段五形成的本地只读产品做系统性异常、生命周期、交付和性能验证，修复验收发现的缺陷，形成可复核的交付报告和后续多平台/多项目基准入口。

## 已确认事实

- 产品固定绑定 `127.0.0.1`，源项目 `.trellis` 保持只读，应用只写自己的注册表和快照。
- 当前运行环境为 macOS，仓库没有 CI、多平台 runner 或测试脚本；项目约定默认不生成测试类。
- 阶段五已经通过基础 fixture、HTTP、SSE、Playwright、四档响应式和只读摘要验收。
- `ProjectRealtimeManager` 已提供可替换 watcher factory、活动监听器计数、运行时状态、轮询降级和幂等关闭入口。
- 阶段六规划明确要求覆盖项目移动/删除、权限异常、坏内容、监听失效、启动恢复、生产启动和运行指标。
- Windows/Linux 的真实文件事件、权限模型和系统外部打开行为无法在当前 macOS 环境中等价实测。
- 已确认跨平台验收采用“macOS 实机完整验证 + Windows/Linux 平台中立代码审查”，报告必须明确未完成 Windows/Linux 实机覆盖。

## 需求

1. 使用临时 fixture 复验扫描、手动登记、无效项目、重复项目和源项目只读边界。
2. 验证焦点项目 Spec/Task/配置变化产生正确的局部失效事件；历史项目变化不产生监听事件。
3. 验证服务重启只恢复焦点项目，活动监听器数量与成功恢复的焦点项目数量一致，历史项目保持零监听。
4. 验证项目移动、删除、必需结构缺失、权限不足、坏 YAML/JSON/JSONL/Markdown 和运行期监听错误的诊断、不可用状态与恢复路径。
5. 验证原生监听启动失败或运行期错误会降级为轮询；轮询也失败时释放运行时资源并暴露失败。
6. 验证生产构建、静态资源、健康接口、未知 API、SSE 连接清理、自动打开失败容错和退出信号释放端口。
7. 记录扫描/索引耗时、焦点监听器数量、待处理变化数、事件数量、注册表/快照大小和进程内存等基线指标。
8. 形成中文阶段六验证报告，区分“当前 macOS 实测”“平台中立代码审查”和“尚未在 Windows/Linux 实机验证”。
9. 验收过程中发现的缺陷直接修复并重新执行相关回归；不扩大到安装包、远程访问或应用内编辑。
10. 不默认新增仓库测试类；验证优先使用 `/tmp` fixture、临时脚本、HTTP/进程检查和 Playwright。

## 验收标准

- [x] 阶段六清单中的扫描、登记、焦点/历史、重启恢复、异常诊断和生产退出均有可复核结果。
- [x] 焦点监听器计数与注册表焦点集合一致，历史项目变化不会产生实时事件。
- [x] 项目删除、移动、权限或结构异常不会遗留监听器，也不会覆盖最后成功快照。
- [x] 原生监听与轮询降级路径均被受控验证，资源释放结果明确。
- [x] API、SSE 和页面不暴露堆栈，不出现未知控制台错误或越界文件访问。
- [x] 源项目只读摘要在非故障注入文件之外保持一致。
- [x] 生产构建与启动通过，退出后端口可立即重新绑定。
- [x] 输出指标基线和阶段六验证报告，并明确真实平台覆盖边界。
- [x] `pnpm lint`、`pnpm typecheck`、`pnpm build`、`git diff --check` 全部通过。

## 验证发现

1. `bad-content` 登记返回的 `config-yaml-invalid`、`task-json-invalid`、`task-jsonl-invalid` 诊断直接包含 YAML/JSON 解析器英文错误，违反用户可见中文反馈约定。
2. 焦点项目缺少 `config.yaml` 后会正确进入 `unavailable`、停止监听并保留最后快照；恢复文件后调用刷新虽清除了错误，但项目状态仍为 `unavailable`，页面无法通过“重新校验”恢复为可用历史项目。

## 实施结果

- 已使用 `/tmp/trellis-visual-console-phase6-b4fvqn` 完成 28 个有效候选、无效结构、坏内容和受控 watcher 验证。
- 已修复解析诊断中文化和不可用项目成功刷新后的状态恢复问题，并完成 API 与 Playwright 回归。
- 已输出 `docs/validation/phase-6-report.md`，记录环境、场景、指标、缺陷与平台覆盖边界。
- macOS 已完成实机覆盖；Windows/Linux 仅完成路径、应用数据目录、Chokidar 和外部打开的平台中立代码审查，未声明实机通过。

## 不在本阶段范围内

- Electron/Tauri、安装包、签名、公证和自动更新。
- 云端、远程访问、多用户、认证、数据库、WebSocket 和 Core SDK。
- 应用内编辑、搜索、导出、图表或 Runtime 管理页面。
- Windows/Linux CI 基础设施、实机 runner 或长期性能监控服务。
