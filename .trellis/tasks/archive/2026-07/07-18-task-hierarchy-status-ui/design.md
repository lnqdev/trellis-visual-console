# 任务层级与状态展示技术设计

## 1. 设计目标

在不改变 Trellis 源项目、不引入新依赖和全局状态库的前提下，将现有 Task 平铺列表升级为可折叠树，并完整表达标准状态、直接子任务进度、跨活动/归档关系和祖先导航。

## 2. 数据边界

### 2.1 关系事实来源

- 以父任务 `children` 为主要关系来源，使用子任务 `parent` 做反向校验与缺失关系补充。
- `subtasks` 不参与层级构建。
- 关系引用使用任务目录名解析，但快照和 API 对外输出已解析的 `sourcePath`，前端不得再按目录名猜测。
- 同时加载活动与归档任务后再做两阶段关系解析，保证活动父任务能够关联已归档子任务。

### 2.2 快照与 API 合同

为 `TaskSummarySnapshot` 和 `TaskSummaryApi` 增加：

- `parentSourcePath: string | null`
- `childSourcePaths: string[]`

集合归属继续由快照的 `tasks.active / tasks.archived` 容器表达，避免重复字段与旧快照产生不一致。存储 Schema 对新增关系字段提供兼容默认值，使现有版本 1 快照仍可读取；重新索引后写入完整关系。API 继续通过共享 Zod Schema 单点校验，不暴露原始 `task.json` 记录。

### 2.3 两阶段索引

1. 第一阶段读取每个任务的基础摘要，以及仅供索引器使用的原始 `parent`、`children` 目录名。
2. 合并活动与归档任务，建立“目录名 -> 任务摘要”索引。
3. 收集全部候选边，按“双向一致 > 仅父声明 > 仅子声明”的确定性优先级解析，不能依赖目录枚举顺序。
4. 将关系投影为 `parentSourcePath / childSourcePaths`，关系取舍完成后按父任务原始 `children` 恢复子任务顺序。
5. 检测自引用、循环、重复子引用、缺失引用和互相矛盾的双向引用；忽略危险边并生成稳定中文诊断。

关系损坏只影响对应边，不丢弃任务摘要，也不阻止其他任务展示。

## 3. 前端派生模型

`TaskBrowser` 接收活动与归档两个完整集合，在组件内用 `useMemo` 建立任务索引、集合内树、祖先路径和直接子任务进度。

进度分母是可解析的全部直接子任务；子任务属于归档集合，或状态为 `completed / done` 时计入完成。Tab 数量始终使用原始集合数组长度。

## 4. 交互设计

### 4.1 树形列表

- 父任务行具有独立展开按钮，任务主体按钮继续负责选中。
- 首次进入时默认展开包含未完成子任务的祖先；手动折叠状态在组件会话内保存。
- 活动与归档分别维护展开集合。
- 视觉缩进最多三级；更深节点使用封顶缩进、延续树线并显示深度标记。
- URL 或面包屑选中深层任务时，自动展开所有可解析祖先。

### 4.2 跨集合关系

- 活动树只直接渲染活动任务节点，归档树只直接渲染归档任务节点。
- 活动父任务存在归档子任务时，在其展开区域显示“N 个子任务已归档”按钮。
- 点击后切换到归档 Tab，并设置组件内归档聚焦父任务，只展示该父任务关联的归档子任务。
- 聚焦栏显示来源父任务和清除按钮；清除后恢复完整归档树。
- 归档子任务行显示父任务标题作为上下文。

### 4.3 状态与详情

- 状态图标使用 Lucide 的圆形、进行、查看、完成和未知语义图标；图标配中文文字，颜色只作辅助。
- 父任务行显示紧凑的 `已完成数/总数`，不引入宽胶囊或进度图表。
- 详情头部增加可点击祖先面包屑；当前任务为末级不可点击文本。
- 面包屑导航复用任务选择入口，确保 Tab、展开状态、URL 和文档选择同步。

## 5. 可访问性与响应式

- 展开按钮使用 `aria-expanded` 和包含任务名的可访问名称；树使用合适的树或等价列表语义。
- 状态文字对屏幕阅读器可见，不能只依赖颜色或图标。
- 归档聚焦栏使用清晰按钮，不依赖悬停操作。
- 300px 桌面侧栏保持现有宽度；窄屏堆叠布局下树行允许标题省略，但状态与展开按钮不能相互遮挡。

## 6. 兼容与回滚

- 不修改 Trellis `task.json`，不新增写接口。
- 旧快照缺少关系字段时按无层级平铺显示；显式刷新后获得完整关系。
- 未知状态保留原始文本；旧 `done` 映射为“已完成”并计入完成进度。
- 回滚时删除新增关系字段和树形渲染即可恢复平铺列表，现有任务详情与文档 API 不受影响。

## 7. 风险控制

- 循环关系可能导致递归渲染或祖先查找死循环：索引和前端派生均使用访问集合设限。
- 跨集合选择可能造成 Tab 与 URL 不一致：所有导航统一通过一个按 `sourcePath` 判定集合的入口。
- 旧持久化快照可能因严格 Schema 拒绝新增字段：新增字段必须设置向后兼容默认值并验证旧快照读取。
