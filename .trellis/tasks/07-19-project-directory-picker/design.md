# 添加项目支持本机目录选择：技术设计

## 设计目标

在不引入桌面壳、不改变现有扫描与登记流程的前提下，让 Windows 和 macOS 用户能够从“添加 Trellis 项目”页面打开系统目录选择对话框，并把所选绝对路径回填到指定输入框。

## 方案选择

采用“本地 Node 服务桥接操作系统目录选择器”的方案：

- Windows 通过 PowerShell 调用 `System.Windows.Forms.FolderBrowserDialog`。
- macOS 通过 `osascript` 执行 `choose folder`。
- Linux 本次不实现原生目录选择，接口返回稳定的不支持错误，页面保留手工输入能力。

不采用浏览器 `showDirectoryPicker()`，因为浏览器不会提供可直接交给本地服务扫描的真实绝对路径。不采用 Electron 或 Tauri，因为桌面打包、签名和发布体系超出本次需求。

## 模块边界

### 共享 API 合同

在 `src/shared/api.ts` 中新增目录选择响应 Schema 和类型。响应使用可判别联合表达两种正常结果：

- `{ status: "selected", path: string }`
- `{ status: "cancelled" }`

取消属于正常用户行为，不作为异常响应。平台不支持、系统命令不可用或系统对话框启动失败继续使用统一 API 错误响应。

### 本地服务

新增独立的系统目录选择模块，职责仅包括：

- 根据 `process.platform` 选择 Windows 或 macOS 适配器。
- 使用 `execFile` 或等价的参数化进程调用，不通过 shell 拼接路径或用户输入。
- 将系统输出规范化为“已选择”或“已取消”。
- 将平台不支持、命令不可用和非取消失败转换为可识别错误。
- Windows 显式设置 UTF-8 输出编码，确保包含中文的目录路径可以正确返回。

目录选择模块通过独立的系统路由注册 `POST /api/system/directories/select`，不挂载到现有 `ProjectApiService`。接口只接受严格 JSON `{}`，避免跨站简单请求直接触发本机对话框；该接口本身不扫描、登记或读取所选目录。

### Web 客户端

在 `src/web/api-client.ts` 新增目录选择请求函数，并在响应边界使用共享 Schema 校验。`useProjectConsole` 暴露对应动作，`App` 将动作传给 `ProjectDiscovery`，保持组件不直接绕过现有状态与 API 边界。

`ProjectDiscovery` 在两个路径输入框旁增加文件夹图标按钮：

- 快速扫描按钮回填 `scanRoot`。
- 手动添加按钮回填 `manualPath` 后立即调用现有单项目登记动作。
- 扫描根目录选择成功只回填，不自动扫描；单项目目录选择成功自动校验登记并刷新项目列表。
- 取消后保持原值和当前提示不变。
- 目录选择期间禁用两个选择按钮，防止重复触发。
- 失败时显示中文局部提示，用户可以继续手工输入或再次选择。

## 数据流

1. 用户点击某个路径输入框旁的目录选择按钮。
2. 页面记录本次选择目标，并调用本地目录选择 API。
3. 本地服务按当前操作系统启动原生目录对话框。
4. 用户确认时，服务返回绝对路径；用户取消时，服务返回 `cancelled`。
5. 页面仅在 `selected` 时更新对应输入框。
6. 扫描根目录等待用户主动点击“扫描”；单项目根目录立即调用现有登记接口，成功后刷新并选中项目。

## 错误处理

- Windows 或 macOS 用户取消对话框：返回 `cancelled`，不显示错误。
- Linux 调用：返回 HTTP 501 和稳定错误码 `directory-picker-unsupported`。
- 原生命令不存在或无法启动：返回 HTTP 500 和稳定错误码 `directory-picker-unavailable`。
- 已有目录选择仍在进行时再次请求：返回 HTTP 409 和稳定错误码 `directory-picker-busy`。
- 服务端日志只记录平台、错误类型和稳定错误码，不记录用户选择的绝对路径。
- 页面优先展示 API 返回的中文错误信息，并保持输入框原值。

## 并发与生命周期

目录选择模块在服务进程内只允许一个活动对话框。活动请求完成、取消或失败后必须释放占用状态，避免多标签页或重复点击同时打开多个系统窗口。

原生子进程随请求完成，不引入常驻后台进程。服务关闭时不增加额外持久化状态。

## 兼容性与安全

- 服务继续绑定 `127.0.0.1`，不新增远程文件系统访问能力。
- 选择路径本身不触发读取；扫描和登记仍通过现有路径校验与项目结构校验。
- 不修改注册表、快照或 Trellis 项目文件。
- 手工输入始终保留，Linux 和原生对话框失败场景仍可完成原有流程。
- 不把输入框内容拼接进 PowerShell 或 AppleScript 命令。

## 验证

- 执行 `pnpm lint`。
- 执行 `pnpm typecheck`。
- 执行 `pnpm build`。
- 在 Windows 实机验证两个入口的打开、选择、取消、中文路径回填和重复点击保护。
- 对 macOS 命令参数、取消分支和 UTF-8 路径处理进行代码审查；当前 Windows 环境不声明 macOS 实机通过。
- 使用浏览器自动化验证选择结果回填后的扫描和登记页面流程；系统原生对话框部分使用 Windows 应用控制完成实机验证。

## 回滚

删除新增目录选择 API、系统适配器和两个图标按钮即可恢复为纯手工输入。现有项目数据与业务接口不需要迁移或回滚。
