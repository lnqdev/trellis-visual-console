# 添加项目支持本机目录选择

## Goal

用户在“添加 Trellis 项目”页面填写本机路径时，可以打开操作系统目录选择对话框并将所选绝对路径自动回填，减少手工复制路径的操作和输入错误。

## Background

- 当前添加页包含“快速扫描”的扫描根路径和“手动添加”的项目根路径两个输入框。
- 两个输入框当前都只支持手工输入或粘贴绝对路径。
- Web 页面只通过 `/api` 访问本地 Node 服务，不能直接读取本机文件系统。
- 浏览器原生目录选择能力不会向页面提供可交给本地服务使用的真实绝对路径，因此需要由本地服务调用系统目录选择对话框。

## Requirements

- 保留路径输入能力，目录选择作为输入辅助，不改变现有扫描与登记流程。
- 用户点击目录选择按钮后，由本地服务打开操作系统目录选择对话框。
- 用户确认选择后，页面将返回的绝对路径回填到对应输入框。
- 用户取消选择时，不清空或覆盖输入框现有内容，也不显示错误提示。
- 目录选择进行中时避免重复触发，并为失败场景提供中文反馈。
- 前后端通过 `src/shared` 中的共享 Schema 定义并校验目录选择 API 合同。
- 不扩大现有源项目读取权限；只有用户最终执行扫描或登记时，才复用现有项目校验与访问逻辑。
- 用户可见文案和新增代码注释使用中文。

## Scope Decision

- “快速扫描”的扫描根路径和“手动添加”的项目根路径两个输入框都提供目录选择入口。
- Windows 和 macOS 使用原生系统目录选择对话框。
- Linux 本次不实现原生目录选择，继续支持手工输入；用户触发目录选择时显示明确的中文不支持提示。

## Acceptance Criteria

- [ ] 用户可以从添加项目页面打开本机目录选择对话框。
- [ ] 确认选择后，所选绝对路径自动回填到目标输入框。
- [ ] 取消选择不会修改输入框已有值，也不会产生失败提示。
- [ ] 选择失败时页面显示可理解的中文错误信息，并允许再次尝试。
- [ ] 现有手工输入、扫描候选、批量登记和单项目登记行为保持不变。
- [ ] Windows 和 macOS 调用各自的系统目录选择对话框；Linux 不影响手工输入，并提供不支持提示。
- [ ] API 响应在前端边界通过共享 Schema 校验。
- [ ] `pnpm lint`、`pnpm typecheck`、`pnpm build` 全部通过。

## Out Of Scope

- 将应用迁移为 Electron、Tauri 或其他桌面壳。
- 改变 Trellis 项目的结构校验、扫描深度、登记规则或持久化格式。
- 默认扫描用户未明确选择或输入的目录。
